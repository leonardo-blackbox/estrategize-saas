Create `backend/src/services/ragService.ts` with a single exported function `retrieveRAGContext`:

```typescript
export interface RAGChunk {
  id: string;
  document_id: string;
  content: string;
  token_count: number;
  metadata: Record<string, unknown>;
  similarity: number;
}

export async function retrieveRAGContext(
  userMessage: string,
  consultancyId: string,
): Promise<RAGChunk[]>
```

Implementation steps:
1. Import `generateEmbeddings` from `./embeddingService.js` and `supabaseAdmin` from `../lib/supabaseAdmin.js`
2. Call `generateEmbeddings([userMessage])` to get the query vector (take `[0]` from result array)
3. Convert the vector to a string format for PostgREST: `'[' + vector.join(',') + ']'` (same pattern used in Phase 10 for storing embeddings — PostgREST needs string representation for vector type)
4. Call `supabaseAdmin.rpc('match_knowledge_chunks', { query_embedding: vectorString, match_threshold: 0.5, match_count: 5, filter_scope: 'global', filter_consultancy_id: null })` for global chunks
5. Call `supabaseAdmin.rpc('match_knowledge_chunks', { query_embedding: vectorString, match_threshold: 0.5, match_count: 5, filter_scope: 'consultancy', filter_consultancy_id: consultancyId })` for consultancy chunks
6. Run both RPC calls in parallel via `Promise.all`
7. Merge results, deduplicate by `id` (use a Map), sort by `similarity` descending, take top 5
8. Wrap the entire function body in try/catch — on error, `console.error('RAG retrieval failed:', error)` and return empty array `[]` (graceful degradation: chat works without RAG if retrieval fails)
9. Use threshold 0.5 (not 0.7 default) to be more inclusive — the top-5 selection handles relevance filtering

Important:
- Do NOT use `ensureAdmin()` helper pattern from consultancyAIService — just import supabaseAdmin directly and null-check at the top: `if (!supabaseAdmin) return [];`
- The function must be self-contained in ragService.ts — no dependencies on consultancyAIService
