Create `backend/src/routes/consultancyDocuments.ts` as a new Express Router mounted at `/api/consultancies/:consultancyId/documents`.

Use `requireAuth` middleware (NOT requireAdmin — these are member routes).

**CRITICAL ownership check:** Every route must first call `getConsultancy(req.userId, consultancyId)` from `consultancyService.ts`. If null, return 404. This ensures RLS + app-level check that user owns the consultancy.

**Constants and multer config:** Mirror exactly from `admin/knowledge.ts`:
- ALLOWED_MIME_TYPES: `['application/pdf', 'text/plain', 'text/markdown']`
- MAX_FILE_SIZE: `10 * 1024 * 1024` (10 MB)
- multer memoryStorage with same fileFilter (including `.md`/`.txt` octet-stream fallback)
- `resolveFileType` helper (same logic)

**Route: GET /** — List documents for consultancy
- Extract `consultancyId` from `req.params.consultancyId`
- Verify ownership via `getConsultancy(req.userId, consultancyId)`
- Call `getDocumentsByScope({ scope: 'consultancy', consultancyId, userId: req.userId })`
- Return JSON array

**Route: POST /** — Upload document to consultancy
- Extract `consultancyId` from `req.params.consultancyId`
- Verify ownership via `getConsultancy(req.userId, consultancyId)`
- multer upload.single('file') with same error handling pattern as admin/knowledge.ts
- Insert into knowledge_documents with `scope: 'consultancy'`, `consultancy_id: consultancyId`
- Return 201 with document (status: 'processing')
- Fire background IIFE: parseFile + chunkText + generateEmbeddings + insert chunks
- Chunk metadata: `{ scope: 'consultancy', consultancy_id: consultancyId, document_name: originalname }`
- On error: update document status to 'error' with error_message

**Route: DELETE /:docId** — Delete document from consultancy
- Extract `consultancyId` from `req.params.consultancyId` and `docId` from `req.params.docId`
- Verify ownership via `getConsultancy(req.userId, consultancyId)`
- Call `deleteDocument(docId, req.userId)` (service verifies user_id match)
- Return 204 on success, 404 if not found

**Router params:** Use `Router({ mergeParams: true })` since consultancyId comes from parent mount.
