Use Express Router with requireAuth + requireAdmin middleware (same pattern as courses.ts).

ALLOWED_MIME_TYPES: `['application/pdf', 'text/plain', 'text/markdown']`
MAX_FILE_SIZE: `10 * 1024 * 1024` (10 MB)

Configure multer with memoryStorage, fileFilter for allowed MIME types, and limits.fileSize.

Map MIME types to fileType: `'application/pdf' -> 'pdf'`, `'text/plain' -> 'txt'`, `'text/markdown' -> 'md'`. Also accept `.md` and `.txt` extensions as fallback when MIME is `application/octet-stream`.

Three routes:

1. `GET /` — Calls `getDocumentsByScope({ scope: 'global' })`. Returns `res.json(documents)`.

2. `POST /` — Async model (returns immediately with status 'processing'):

   Uses `upload.single('file')`. Extracts `req.file`, determines `fileType` from MIME/extension.

   **Step A — Insert document record directly via supabaseAdmin:**
   ```typescript
   import { supabaseAdmin } from '../../lib/supabase.js';

   const { data: document, error: insertError } = await supabaseAdmin
     .from('knowledge_documents')
     .insert({
       user_id: userId,
       scope: 'global',
       name: req.file.originalname,
       file_type: fileType,
       file_size_bytes: req.file.buffer.length,
       status: 'processing',
     })
     .select()
     .single();

   if (insertError || !document) {
     return res.status(500).json({ error: insertError?.message ?? 'Failed to create document' });
   }
   ```

   **Step B — Return 201 immediately with the processing document:**
   ```typescript
   res.status(201).json(document);
   ```

   **Step C — Fire background processing AFTER response is sent:**
   ```typescript
   // Background: parse, chunk, embed, update status
   (async () => {
     try {
       const text = await parseFile(fileType, req.file!.buffer);
       const chunks = chunkText(text);
       if (chunks.length === 0) throw new Error('Document produced no chunks');

       const embeddings = await generateEmbeddings(chunks.map(c => c.content));

       const chunkRows = chunks.map((chunk, i) => ({
         document_id: document.id,
         chunk_index: chunk.index,
         content: chunk.content,
         token_count: chunk.tokenCount,
         embedding: `[${embeddings[i].join(',')}]`,
         metadata: { scope: 'global', consultancy_id: null, document_name: req.file!.originalname },
       }));

       const { error: chunksError } = await supabaseAdmin.from('knowledge_chunks').insert(chunkRows);
       if (chunksError) throw new Error(`Failed to insert chunks: ${chunksError.message}`);

       await supabaseAdmin
         .from('knowledge_documents')
         .update({ status: 'ready', chunk_count: chunks.length })
         .eq('id', document.id);
     } catch (err) {
       const message = err instanceof Error ? err.message : String(err);
       console.error(`[knowledge] Background processing failed for doc ${document.id}:`, message);
       await supabaseAdmin
         .from('knowledge_documents')
         .update({ status: 'error', error_message: message })
         .eq('id', document.id);
     }
   })();
   ```

   Key: the IIFE runs after `res.status(201).json(document)` has already been sent. Do NOT await it. The `parseFile`, `chunkText`, `generateEmbeddings` are imported from knowledgeService (exported in the pre-step).

   Do NOT call `processDocument()` — it would try to insert a duplicate record. Use the individual helpers directly.

3. `DELETE /:id` — Validates `req.params.id` is present. Calls `deleteDocument(id, req.userId)`. If returns false, respond 404. If true, respond 204.
