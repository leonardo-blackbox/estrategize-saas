**KnowledgeUpload.tsx** (~70 lines) — Micro-module:

Props: `{ onUpload: (file: File) => void; isUploading: boolean }`

- File input (hidden) accepting `.pdf,.txt,.md`
- Dropzone div with dashed border (`border-dashed border-[var(--border-hairline)]`), click triggers file input
- Drag-and-drop: `onDragOver` (prevent default, highlight), `onDrop` (prevent default, extract file, call onUpload)
- Shows "Arraste um arquivo ou clique para selecionar" text
- Accepted formats note: "PDF, TXT ou MD. Maximo 10 MB"
- While `isUploading`, show spinner and "Processando..." text, disable input

**KnowledgeList.tsx** (~80 lines) — Micro-module:

Props: `{ documents: KnowledgeDocument[]; onDelete: (id: string) => void; isLoading: boolean }`

- If `isLoading`: show 3 skeleton rows (pulse animation)
- If `documents.length === 0` and not loading: empty state — "Nenhum documento adicionado. Faca upload de PDFs da metodologia Iris para treinar a IA."
- For each document, render a row/card with:
  - File icon (different for pdf/txt/md)
  - Document name (`name`)
  - File size formatted (KB/MB)
  - Chunk count (`chunk_count chunks`)
  - Status badge: `processing` = amber "Processando", `ready` = emerald "Indexado", `error` = red "Erro"
  - If status is `error`, show `error_message` in small red text below
  - Delete button (trash icon) — calls `onDelete(doc.id)`. Disabled while status is `processing`
  - Date (`created_at`) formatted relative or dd/mm/yyyy
